# TODO — 作業タスク管理

このファイルは作業タスクを「MUST / SHOULD / WANT」で管理するための公式チェックリストです。
AIエージェントは必ずこのファイルを参照し、下記のルールに従ってください。

---

## 運用ルール（Agent向け）

- **MUST**: 必須の作業。作業完了時にすべてチェックされていなければ作業を中断し、ユーザーに確認してください。
- **SHOULD**: 取り組むべき作業。MUST と同様に、作業完了時には全てチェックを付ける必要があります。
- **WANT**: 任意の追加改善。未チェックでも構いません。
- 新しいタスクを追加する場合、行頭に `[MUST]` / `[SHOULD]` / `[WANT]` を付け、同時に `manage_todo_list` ツールで機械的に反映してください（ツールとファイルの両方を更新します）。
- 作業が完了したら、MUST と SHOULD がすべてチェック済みであることを確認してください。全てチェックされていれば、`MUST` と `SHOULD` のセクションをこのファイルから削除してください（WANT は残して構いません）。未チェックが残っている場合は作業を中断してユーザーに報告してください。
 - 作業が完了したら、MUST と SHOULD がすべてチェック済みであることを確認してください。完了したタスク行だけを削除し、セクション見出し（`MUST` / `SHOULD` / `WANT`）は必ず残してください。未チェックが残っている場合は作業を中断してユーザーに報告してください。

---

## フォーマット

- 各タスクは次の形式で記述してください:
  - `- [ ] [MUST] タイトル — 簡単な説明`
  - `- [x] [SHOULD] タイトル — 完了済みの表示`

## 自動化提案（SHOULD の実施内容）

### 目的
作業完了前に `TODO.md` と `manage_todo_list` を検証・同期し、MUST/SHOULD が全てチェック済みであれば自動的に該当セクションを除去して同期する自動化フローの提案。

### 概要
1. スクリプト: `scripts/sync-todo.ts`（TypeScript、Bun 実行想定）
2. 動作:
  - `TODO.md` をパースして `[MUST]` / `[SHOULD]` タスクを抽出する
  - それらがすべて `[x]`（完了）か検証する。未完了があれば欠落タスク一覧を出力して終了する
  - 完了していれば `manage_todo_list` ツールに完了状態を送信して同期する
  - `TODO.md` から完了済みのタスク行のみを削除してファイルを上書きする
  - 変更をコミット（例: `git commit -m "chore: finalize TODO MUST/SHOULD"`）する。自動コミットは `--yes` フラグが必要
3. 実行:
  - ローカル: `bun run tsx scripts/sync-todo.ts`
  - CI: `bun run sync-todo` を `package.json` に追加

### 注意
- Windows では WSL/Git Bash を推奨
- 自動でコミットする前に差分確認を必須化する

## 管理ツール同期手順（Agent向け）

1. このファイルを編集したら、対応する全てのタスクを `manage_todo_list` ツールに送ってください。タイトルの先頭に `[MUST]`/`[SHOULD]`/`[WANT]` を付けることで、機械的にグルーピングできます。
2. `manage_todo_list` の一覧は常にこのファイルの内容と一致させてください。
3. 作業完了の直前に、MUST と SHOULD が全て `[x]` になっていることを検証し、未チェックがあれば作業を中断してユーザーに確認してください。
